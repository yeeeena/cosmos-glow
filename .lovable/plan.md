# 생성 페이지 입력 영역 UX 개선

## 현재 문제점

- 하단 입력 바에 제품 칩, 텍스트 입력, 옵션 버튼들이 평면적으로 나열되어 역할 구분이 어려움
- "IMAGE", "STUDIO" 같은 라벨이 무엇을 의미하는지 직관적이지 않음
- 제품 이미지 업로드가 명확한 버튼으로 존재하지 않음
- 컨셉/스타일 선택이 별도 영역 없이 섞여 있음

## 개선 방향: 3단계 흐름 기반 입력 UI

하단 고정 바를 **3개의 명확한 단계**로 시각적 구분합니다:

```text
+----------------------------------------------------------+
|  [1. 제품 이미지]   [2. 컨셉 스타일]   [3. 생성 옵션]      |
|                                                          |
|  +---------------+  +--------------+  +------+------+    |
|  | 📷 이미지      |  | ✨ 스타일     |  | 비율 | 품질 |    |
|  | 업로드         |  | 선택          |  +------+------+    |
|  +---------------+  +--------------+  | 버전           |    |
|                                       +----------------+    |
|  [프롬프트 입력창 ...]                     [생성하기 ->]    |
|  Cost 16 credits · 더 많은 실행 횟수 얻기                  |
+----------------------------------------------------------+
```

## 구체적 변경 사항

### 1. 제품 이미지 업로드 영역

- 큰 아이콘 버튼: 카메라/이미지 아이콘 + "제품 이미지 업로드" 텍스트
- 클릭 시 파일 선택 다이얼로그 (input type="file" 연결)
- 업로드된 이미지가 있으면 썸네일 미리보기로 표시 (작은 카드 형태)
- 한장만 업로드 가능

### 2. 컨셉샷 스타일 선택

- 스타일 프리셋을 시각적 카드/칩으로 표시
- 프리셋 예시: "클린 스튜디오", "모델사용", "라이프스타일", "아트디렉팅", "로우키그라데이션"
- 각 칩에 작은 아이콘 포함, 선택 시 파란색(primary) 하이라이트
- ToggleGroup 컴포넌트 활용하여 단일 선택

### 3. 생성 옵션 간소화

- 비율, 버전, 품질을 한 줄의 세그먼트 버튼으로 통합
- 각 옵션은 Popover로 선택지 표시 (현재 OptionPill을 클릭하면 옵션 목록 노출)
- 기존 "IMAGE", "STUDIO" 드롭다운 제거 (역할이 위 1, 2번으로 흡수)

### 4. 프롬프트 입력 + 생성 버튼

- 입력창과 생성 버튼을 한 줄에 배치 (입력창 우측에 생성 버튼 내장)
- placeholder: "추가 설명을 입력하세요 (선택사항)"
- 생성 버튼은 입력창 오른쪽 끝에 아이콘 버튼으로 통합

## 기술적 세부 사항

### 수정 파일

- `**src/pages/CreatePage.tsx**`: 전체 하단 입력 바 재구성
  - 파일 업로드용 hidden input + useRef
  - 업로드된 이미지 상태 관리 (useState로 File[] 또는 미리보기 URL)
  - 스타일 프리셋 데이터 배열 + 선택 상태
  - Popover 기반 옵션 선택기 (비율, 버전, 품질)
  - 레이아웃을 상단 액션 행 + 하단 입력 행 2줄 구조로 변경

### 사용 컴포넌트

- `ToggleGroup` / `ToggleGroupItem` - 스타일 선택
- `Popover` / `PopoverTrigger` / `PopoverContent` - 옵션 선택
- lucide 아이콘: `ImagePlus`, `Palette`, `Settings2`, `Upload`, `Sparkles`

### 상태 관리

- `uploadedImages: string[]` - 업로드된 이미지 미리보기 URL
- `selectedStyle: string` - 선택된 컨셉 스타일
- `ratio: string` - 비율 (기본 "9:16")
- `version: string` - 버전 (기본 "4v")  
- `quality: string` - 품질 (기본 "High")
